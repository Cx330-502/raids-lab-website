## Choose the method of creating a mirror image

At present, we offer**Four kinds**The methods of creating a mirror image are as follows:

1. Python+CUDA Custom build
2. Build based on the existing image
3. Based on Dockerfile Build
4. Based on Envd Build

You can select the construction method you want using the drop-down button shown in the figure, and then click the button to call up the construction form.

![Image](./img/imagebuild-dropdown.webp)

### 1. Python+CUDA Custom build

In order to better meet users' demands for Python And CUDA For the need of version customization, we provide`Python+CUDA Custom build`The construction method of it.

Users can select the corresponding options in the form. Python And CUDA Version. If necessary, users should fill it in according to the specifications APT Software packages and Python requirements With the content, you can easily build an image, and the built image can be applied to Jupyter Interactive assignments and Custom Customized homework. The detailed content of the form can be seen in the figure below.

![Image](./img/imagebuild-python+cuda.webp)

### 2. Build based on the existing image

Considering that users may need to build new images based on existing ones, we have provided`Build based on the existing image`The construction method of it.

This construction method requires you to fill out a form, in which you must select a base image and provide a brief description for the image, which will be displayed as the image's identifier. APT Software packages and Python You can leave the dependency field blank. If you need to install a software package, you can enter the package name, separating multiple packages with spaces. For example, if you need to install curl And tree，Then you can fill it out. `curl tree`。

![Image](./img/imagebuild-package.webp)

If the project you plan to run includes requirements.txt For the file, you can copy its content to Python Install it in the dependency folder, but it's important to note that...pip You may encounter unresolved issues when parsing dependencies. To avoid the failure of image building, you can first try to manually install the dependencies in the container and then fill in the dependencies based on the installation results. The above figure is an example of form filling. After completing the form, you can click on "Start Building" in the lower-right corner of the form to start image building.

### 3. Based on Dockerfile Build

Users with high demands for autonomy can choose to use it.`Based on Dockerfile Build`The construction method of this system can effectively meet users' flexible demands for images.

If you choose this build method, you only need to fill in an image description and a complete and grammatically correct one. Dockerfile（Given the domestic network issues, it may be necessary to switch to another source during the corresponding download and code retrieval processes. Both of these steps are mandatory, and a possible example is shown in the figure below.

![Image](./img/imagebuild-dockerfile.webp)

The example used here Dockerfile See below.

```dockerfile
FROM gpu-harbor.act.buaa.edu.cn/user-liuyizhou/nvidia-pytorch:24.12-v1.2.1

USER root

RUN pip install onnxruntime-gpu && \
    echo -e '#!/bin/bash\nunzip -o /home/$NB_USER/workspaces/test.zip -d /workspace/test' >> /usr/local/bin/before-notebook.d/prepare_workspace.sh && \
    chmod +x /usr/local/bin/before-notebook.d/prepare_workspace.sh
```

You can start the image creation by clicking on "Start Creating" in the lower-right corner of the form.

## View the mirror image

After you click on "Start Making", you will be able to see the image you submitted for making in the list at the bottom of the page. The initial status should be "Waiting".

![Image](./img/imagebuild-result.webp)

You can view the detailed information by clicking on the description of the image. At this point, you will be able to see the detailed log of the image build under the "Build Log" tab, which will help you troubleshoot any potential issues.

![Image](./img/imagebuild-log.webp)

In addition, you can also switch the tab to "Dockerfile”Or "Envd Click on "Configuration" to view the forms that the system has generated for you or that you have filled out. Dockerfile Or Envd Script.

![Image](./img/imagebuild-dockerfile-show.webp)

Finally, if your image build is successful, you will be able to see the status of the image change to "Success" on that page, and you will be able to see the size of the completed image. Please note that due to Docker The hierarchical storage mechanism used means that the actual storage usage may be much smaller than the total size of all the images. At the same time, you can also see the type of the image you just created in the "Image List".Custom”Its mirror image, you will be able to use it to start "batch processing jobs" or "interactive jobs".

![Image](./img/imagebuild-list.webp)
