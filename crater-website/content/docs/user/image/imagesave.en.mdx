# Mirror saving

When the user is on Jupyter A new device has been installed in the environment. Python After packaging, if you stop the operation, these files will be deleted by default. Python The package will be lost.

For this reason, we provide a mirror saving function, which allows users to directly save their current changes as a mirror, making it more convenient to use next time.

<Callout>

Due to the fact that the images of deep learning are usually very large, and factors such as the network may cause interference during the process, it's difficult to guarantee the success rate of image preservation. We still recommend that you try to do it as much as possible through Dockerfile Or create a mirror using low-code methods. We might impose the following restrictions:

1. The total size of the images that users can upload
2. The number of mirrors that users can save (in order to save new ones, they must delete the old ones)

Summary: We highly recommend that you use Dockerfile Or create a mirror image using the low-code method!

</Callout>
