# ミラー保存

ユーザーが < で検索する場合、検索結果は次のように表示されます。 Jupyter 環境内に新しいものが設置されました。 Python パッケージ化後、作業を停止すると、デフォルトではこれらが Python パッケージは失われてしまうでしょう。

そのため、当社ではミラー保存機能を提供しています。ユーザーが現在行っている変更内容をミラーとして直接保存でき、次回使用する際に便利です。

<Callout>

ディープラーニングのイメージは通常大きく、ネットワークやその他の要因によって中断されることがあるため、イメージの保存成功率を保証するのは難しいのです。それでも、できるだけ以下の方法で保存することをお勧めします。 Dockerfile または、低コード方式でミラーを作成する方法もあります。ただし、以下のような制限がある場合があります。

1. ユーザーはミラーの総サイズをアップロードできます
2. ユーザーが保存できるミラーイメージの数（新しいものを保存するには、古いものを削除する必要がある）

まとめ：私たちは、あなたが<を通じてそれを行うことを強くお勧めします。 Dockerfile または、低コード方式でミラーを作成する！

</Callout>

## 起動 Jupyter 例

参考にしてください。 [インタラクティブな課題](../quick-start/interactive.md) 起動 Jupyter Lab。

その後、このコンテナー内でいくつかの操作を実行できるかもしれません。例えば、アプリケーションをインストールするなどです。 Pip バッグ：

```shell
# 使用する aliyun の Pip ソース
sudo pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/

# 任意のパッケージをインストールする。
sudo pip install jieba
```

## イメージ内の変更を保存する。

で Jupyter インタラクティブページで、見つけてください。 Crater アイコンのフローティングボール：

![alt text](./img/floatball.webp)

保存画像をクリックし、確認を行ってください。

Crater プラットフォームはあなたをミラー作成ページにリダイレクトし、そこではミラーパッケージングタスクの具体的な進歩状況を確認できます。成功した場合、ミラーを選択する際に次のような説明が表示されます。 `Snapshot of XXXX` のミラーサイト。
