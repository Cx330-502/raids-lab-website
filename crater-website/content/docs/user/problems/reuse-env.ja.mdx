## 概要

計算クラスターのアップグレードや移行のプロセスでは、環境の移行が重要なステップです。この記事では、既存の環境をどのように移行するかをご案内します。 Slurm クラスター環境を新しいコンピューティングプラットフォームに移行させます。主な2つの方法について重点的に説明します。1つは既存のリソースを直接再利用する方法、もう1つは新しいリソースを追加して既存のリソースを置き換える方法です。 Conda 環境と既存の環境に基づいて再建する。 Python 環境。

## 1. 既存のものを再利用する。 Conda 環境

<Callout title="警告" type="warn">
既存のものを直接再利用できるのは確かに便利ですね Conda 環境ですが、以下の要素を慎重に考慮することをお勧めします。

- **性能に影響する**：で Docker コンテナ内に再インストールする。 Python 環境は通常、より優れた実行効率を提供します
- **互換性の問題**：環境をそのまま再利用すると、パス依存の問題が発生する可能性があります。
- **環境の清潔度**：現在の環境には、もはや使用されていない依存パッケージが含まれている可能性があります。

</Callout>

1. **ルート調整**：
   マウントパスを変更して、<を追加してください。 `/home/{username}` に置き換える `/home/LAB/{username}`，整列にする Slurm クラスターで、パスのハードコーディングの問題を解決します。再利用環境が不要な場合は、このステップをスキップできます。

   ![mount](./img/mount.webp)

2. **環境活性化**：
   以下のようなコマンドを実行して、既存の環境をアクティブにします。

   ```bash
   export USERNAME=liyilong
   export PATH=/home/LAB/${USERNAME}/miniconda3/bin:${PATH}
   source /home/LAB/${USERNAME}/miniconda3/etc/profile.d/conda.sh
   export PATH=$CONDA_DIR/bin:${PATH}
   ```

3. **検証環境**：
   以下のコマンドを使用して、環境が正常にアクティブになっているかどうかを確認してください。

   ```bash
   conda activate
   ```

## 2. 既存の環境に基づいて再建する。 Python 環境

<Callout title="優れた点" type="info">

- **環境が清潔**：もはや使用しない依存関係を削除できます
- **パフォーマンス最適化**：通常、新しいインストールの方が優れたパフォーマンスを発揮します。
- **メンテナンス性**：明確な依存関係のリストは、後続のメンテナンスが容易になります。

</Callout>

1. **生成 Requirements ファイル**：
   使用する pipreqs プロジェクトのカタログをスキャンし、正確な依存関係のリストを生成します。

   ```bash
   pip install pipreqs
   pipreqs /path/to/project --encoding=utf8 --force
   ```

2. **依存関係のリストを最適化する。**：

   - 生成されたものを手動でチェックする。 requirements.txt ファイル
   - 使用しなくなった依存関係を削除する。
   - 固定のキー依存関係のバージョン番号

3. **新しい環境を作成する。**：ミラーリングの製作に関するセクションを参照してください。さまざまな方法で新しい環境を作成できます。

## 結論

私たちは、初期設定に時間がかかるものの、長期的なメンテナンス性とパフォーマンスが向上するため、第二の方法（環境の再構築）を優先的に推奨します。時間的に余裕がある場合は、第一の方法（環境の再利用）を一時的な解決策として利用できますが、その後も環境の再構築をお勧めします。

> 注意：環境を移行する際には、バックアップを必ず実施し、すべての操作手順を記録しておいてください。そうすれば、問題が発生した場合にすぐにロールバックできます。
