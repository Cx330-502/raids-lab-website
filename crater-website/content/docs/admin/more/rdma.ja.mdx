正式に始める前に、いくつかのことを確認しましょう。 RDMA 関連する基本知識：

- **RDMA**：これはオペレーティングシステムのカーネルをバイパスするネットワーク通信技術であり、その中心はネットワークカードを通じてリモートメモリに直接アクセスし、従来の方法を回避することにあります。 TCP/IP プロトコルスタックのデータコピーとコンテキスト切り替えのオーバーヘッド。
- **NVIDIA GPU Direct**[^2]：実現する GPU ビデオメモリとネットワークカード DMA エンジンの直接接続時、 GPU 遠隔ノードと通信する必要がある場合、データは直接送信できます。 InfiniBand または RoCE ネットワークカードでデータを送信する場合、ホストのメモリを経由して中継する必要はありません。
- **ネットワーク仮想化**：Macvlan そして SR-IOV これらは2つの一般的なネットワーク仮想化ソリューションです。Macvlan コンテナーに仮想ネットワークカードインターフェイスを作成することを許可し、物理ネットワーク上で独立したデバイスとして表示させることができます。そして SR-IOV 物理ネットワークカードのハードウェア仮想化機能を利用して、単一の物理機能を（PF）複数の仮想機能に分類される（VF），それぞれの VF いずれも直接割り当てることができます Pod 使用。
- **技術的なアプローチ**：現在 RDMA 主なものは InfiniBand そして RoCE 2つの実現方法[^6]。InfiniBand ネイティブサポート RDMA このプロトコルでは、専用交換機とサブネットマネージャーが必要で独立ネットワークを構築する必要があり、コストが高くなります。しかし、 RoCEv2 そして、従来のイーサネットインフラストラクチャに基づいて、 PFC そして ECN フロー制御メカニズムによる無損失伝送を保証し、インターネット企業で広く利用されています。

私たちの実験室では InfiniBand プランに関しては、まず関連する機器をチェックする必要があります。 IB 情報：

### 1. シングルノードテスト中 InfiniBand 関連情報

まずホストマシンでテストを実施し、クラウドに移行する前にこれらのマシンをテストします。 IB いずれも通用です：

```bash
$ ibdev2netdev
mlx5_0 port 1 ==> ibxxxxxx0 (Up)
mlx5_1 port 1 ==> ibxxxxxx1 (Up)

$ ibstat
CA 'mlx5_0'
        Port 1:
                Link layer: InfiniBand
CA 'mlx5_1'
        Port 1:
                Link layer: InfiniBand
```

- **Up**: それを示す InfiniBand ポートが正常にアクティブ化され、ネットワークに接続されました。
- **Down**: それを示す InfiniBand ポートがアクティブでないか、ネットワーク接続ができない。

### 2. 使用する Ansible ノードのネットワークカードを一括で表示する。

グループ分け：

```toml
[ib-v100]
xx.xx.xx.[xx:xx]

[ib-a100]
xx.xx.xx.[xx:xx]
```

バッチクエリスクリプトの作成：

```yaml
---
- name: Run ibdev2netdev on InfiniBand hosts
  hosts: ib-v100,ib-a100
  gather_facts: no

  tasks:
    - name: Execute ibdev2netdev command
      ansible.builtin.command: ibdev2netdev
      register: ibdev_output
      changed_when: false

    - name: Display ibdev2netdev output
      ansible.builtin.debug:
        var: ibdev_output.stdout_lines
```

返り値が長すぎるので、完全なものは掲載しないでおきます。 `ibdev2netdev` 出力結果から、クラスターの2つのノードの InfiniBand 構成が異なります。

#### V100 ノード

```
mlx5_0 port 1 ==> ibxxxxxx0 (Up)
mlx5_1 port 1 ==> ibxxxxxx1 (Up)
```

これらのノードは、それぞれに2ポートのネットワークカードが搭載されています。 IB ネットワークカード、各ポートの最大速度は 100Gbp/s，それぞれに接続してください。 2 台 36 ポートの IB 交換機上で、2 交換機間にもまだあるよね 4 条項 100Gbps インターネットの接続回線。

- 各ノードには2つの独立した InfiniBand ポート (mlx5_0 そして mlx5_1)
- 両方のポートがオンになっています。 Up 状態

#### A100 ノード

```
mlx5_0 port 1 ==> ibxxxx0 (Down/Up)
mlx5_1 port 1 ==> ibxxxxx0 (Up/Down)
mlx5_bond_0 port 1 ==> bond0 (Up)
```

このバッチのマシンはそれぞれにプラグが差し込まれています。 2 ブロック 200Gbps の IB カードを使って 1 個 IB スイッチ間の相互接続です。しかし、すべてのネットワークカードが互換ではないので、各ノードには1枚しかありません。 IB カードが通りません IB このケーブルはスイッチに接続されています。

mlx5_bond_0 それはイーサネットカードですが、たまたまそうなだけです。 Mellanox メーカー製なので、それも表示されています。
